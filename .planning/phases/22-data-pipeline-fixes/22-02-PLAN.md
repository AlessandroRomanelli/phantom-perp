<objective>
Fix bar_volumes to use true per-bar candle volume (PROF-03) and fix index_price to use exchange data with graceful degradation when unavailable (ROBU-05).

Purpose: bar_volumes currently diffs 24h cumulative values producing ~48% negative entries, corrupting VWAP weights. index_price silently equals last_price, making basis calculations meaningless for correlation and regime trend strategies. Both bugs degrade signal quality and cause strategies to fire on garbage data.
Output: Correct bar_volumes from candle data, index_price from exchange API with sentinel-based graceful degradation.
</objective>
