<objective>
Build the foundational data structures and round-trip reconstruction pipeline for the metrics engine.

Purpose: The metrics engine needs to transform raw `AttributedFill` rows (one per exchange fill) into closed `RoundTrip` objects (one per completed trade) before any performance metrics can be computed. This plan implements VWAP aggregation (D-05), FIFO entry/exit pairing (D-03), open position exclusion (D-04), and per-round-trip fee-adjusted P&L (METR-04 foundation). All work is TDD: tests first, then implementation.

Output: `libs/metrics/engine.py` with `OrderResult`, `RoundTrip` frozen dataclasses, `vwap_aggregate()`, `pair_round_trips()`, and `build_round_trips()` functions. Comprehensive unit tests in `tests/unit/test_metrics_engine.py`.

NOTE on direction inference (review concern): The FIFO pairing algorithm uses position-state tracking. The first order for a given (source, instrument) group establishes direction. If the first order is BUY, subsequent SELL orders close it (LONG trade). If the first order is SELL, subsequent BUY orders close it (SHORT trade). A BUY fill is never ambiguous in isolation -- it is always resolved by its position in the FIFO stack relative to prior orders for that group. This is tested explicitly for both LONG and SHORT scenarios plus overlapping entries.
</objective>

    Replace `NotImplementedError` stubs in `libs/metrics/engine.py` with full implementations:

    **vwap_aggregate(fills: list[AttributedFill]) -> OrderResult:**
    - Assert len(fills) > 0
    - Filter out fills with size <= 0 (zero-size guard per review suggestion)
    - total_value = sum(f.price * f.size for f in fills) -- Decimal arithmetic
    - total_size = sum(f.size for f in fills)
    - If total_size == Decimal("0"): raise ValueError("All fills have zero size for order {order_id}")
    - avg_price = total_value / total_size
    - total_fee = sum(f.fee_usdc for f in fills)
    - filled_at = max(f.filled_at for f in fills) -- latest fill timestamp
    - Return OrderResult with fills[0].order_id, fills[0].instrument, fills[0].primary_source, fills[0].side, avg_price, total_size, total_fee, filled_at

    **pair_round_trips(orders: list[OrderResult]) -> list[RoundTrip]:**
    - Use collections.deque as FIFO entry stack
    - Iterate orders chronologically (caller must sort -- build_round_trips does defensive sort)
    - If stack empty or order.side == stack[0].side: push onto stack (new entry or pyramid)
    - If order.side != stack[0].side: pop entry from left (FIFO), create RoundTrip
    - For RoundTrip P&L direction is determined by the ENTRY order's side:
      - If entry.side == "BUY" (LONG): gross_pnl = (exit.avg_price - entry.avg_price) * size
      - If entry.side == "SELL" (SHORT): gross_pnl = (entry.avg_price - exit.avg_price) * size
      - This means a BUY order that closes a short is identified as an EXIT because entry_stack[0].side == "SELL" != "BUY"
    - size = min(entry.total_size, exit.total_size) -- handle size mismatches conservatively
    - total_fees = entry.total_fee + exit.total_fee
    - net_pnl = gross_pnl - total_fees
    - Remaining stack entries are open positions -- excluded per D-04
    - Return list[RoundTrip]

    **build_round_trips(fills: list[AttributedFill]) -> dict[tuple[str, str], list[RoundTrip]]:**
    - Group fills by (primary_source, instrument) using collections.defaultdict(list)
    - For each group: sub-group by order_id, vwap_aggregate each sub-group -> list[OrderResult]
    - Sort OrderResults by filled_at ascending (defensive sort -- ensures chronological FIFO even if input is unordered)
    - Call pair_round_trips(sorted_orders)
    - Store in result dict keyed by (source, instrument)
    - Only include keys where round_trips is non-empty
