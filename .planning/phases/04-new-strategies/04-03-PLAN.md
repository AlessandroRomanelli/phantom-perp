    ```python
    def _simulate_bar_volumes(n_bars: int = 200) -> NDArray[np.float64]:
        """Simulate bar_volumes as np.diff of 24h rolling volume.

        Model: each bar adds some volume, but np.diff of 24h rolling
        means we subtract what rolled off 24h ago.
        """
        # Simulate hourly volume with variation
        np.random.seed(42)
        hourly_vol = np.abs(np.random.normal(1000, 300, n_bars + 24))
        # 24h rolling sum
        rolling_24h = np.convolve(hourly_vol, np.ones(24), mode='valid')
        # bar_volumes = np.diff of rolling 24h
        return np.diff(rolling_24h)
    ```

    Feasibility pass criteria:
    1. After clamping negatives to 0: cumulative volume is positive and growing
    2. VWAP computed with clamped volumes differs from simple average by > 0.05% of price (adds value)
    3. VWAP is less volatile than price (smoother anchor point)

    If standard approach fails all 3 criteria:

    Create `agents/signals/strategies/vwap.py` with `VWAPStrategy` and `VWAPParams`:

    ```python
    @dataclass
    class VWAPParams:
        session_reset_hour_utc: int = 0     # 00:00 UTC for crypto (VWAP-02)
        deviation_threshold: float = 2.0     # Std deviations from VWAP to trigger (VWAP-03)
        min_session_progress: float = 0.2    # Don't signal in first 20% of session (VWAP-04)
        session_conviction_weight: float = 0.3  # How much session progress affects conviction (VWAP-04)
        atr_period: int = 14
        stop_loss_atr_mult: float = 2.0
        take_profit_atr_mult: float = 1.5    # Tighter TP since targeting mean reversion to VWAP
        min_conviction: float = 0.40
        cooldown_bars: int = 10
        portfolio_a_min_conviction: float = 0.70
        lookback_bars: int = 60              # Rolling window for VWAP if no session reset
        use_session_reset: bool = True       # False if using alternative rolling approach (D-07)
        enabled: bool = True
    ```

    VWAP computation logic:
    - If `use_session_reset=True`: find bars since last session reset (using timestamps vs session_reset_hour_utc). Compute VWAP = sum(close[i] * weight[i]) / sum(weight[i]) where weight = max(bar_volume, 0) (clamped). If using alternative approach, weight = volume (24h rolling).
    - If `use_session_reset=False` (alternative D-07): use rolling window of `lookback_bars` with volumes as weights.
    - Deviation = (current_price - vwap) / std(prices_in_session)
    - Direction: LONG if price below VWAP (deviation < -threshold), SHORT if above (deviation > threshold)

    Session time awareness (VWAP-04):
    - session_progress = time_since_reset / session_length (clamped to [0, 1])
    - conviction_boost = session_progress * p.session_conviction_weight
    - Early session (progress < min_session_progress): suppress signals

    Conviction model (2-component):
    - Deviation magnitude (0-0.60): scales with how far beyond threshold
    - Session progress (0-0.30): later in session = more reliable VWAP = higher conviction
    - Base = 0.10

    Per-instrument config (in YAML):
    - Crypto (ETH, BTC, SOL): session_reset_hour_utc=0 (midnight UTC)
    - Equity (QQQ, SPY): session_reset_hour_utc=14 (09:30 ET = 14:30 UTC, approximate to 14)
