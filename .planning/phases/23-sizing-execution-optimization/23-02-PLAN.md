<objective>
Add a fee-adjusted signal filter to the risk engine that rejects trades where estimated round-trip fees exceed the expected edge.

Purpose: The forensic audit found the system pays $4 in fees for every $1 of alpha captured. Trades with insufficient expected edge relative to fees are net-negative from the start. This filter prevents those trades from executing.

Output: New fee filter check in RiskEngine.evaluate(), min_expected_move_pct config field, tests proving rejection and pass-through behavior.
