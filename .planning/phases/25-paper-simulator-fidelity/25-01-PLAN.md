<objective>
Add probabilistic fill model with adverse selection to the paper simulator.

Purpose: Paper mode currently fills 100% of limit orders instantly at the exact limit price. This makes paper P&L unrealistically optimistic. Adding fill probability gated by price proximity and adverse selection on filled prices will produce paper results that better approximate real-world execution.

Output: `PaperSimulatorConfig` dataclass, `_decide_fill()` helper function, YAML config section, wired into `order_processor` task, with full TDD test coverage.
</objective>

      ```python
      def _decide_fill(
          side: OrderSide,
          order_type: OrderType,
          limit_price: Decimal | None,
          mark_price: Decimal,
          cfg: PaperSimulatorConfig,
          rng: random.Random,

      ```
      Logic:
      - MARKET orders or limit_price is None: return (True, mark_price) — no adverse selection
      - mark_price == 0: return (False, Decimal("0"))
      - Compute distance_bps = abs(limit_price - mark_price) / mark_price * 10000
      - proximity_factor = Decimal("1") / (Decimal("1") + distance_bps / Decimal("100"))
      - probability = float(cfg.fill_probability_base * proximity_factor)
      - If rng.random() > probability: return (False, Decimal("0"))
      - Apply adverse selection: bps_factor = cfg.adverse_selection_bps / Decimal("10000")
      - BUY: effective_price = limit_price * (1 + bps_factor)
      - SELL: effective_price = limit_price * (1 - bps_factor)
      - Quantize to Decimal("0.01") with ROUND_HALF_UP
      - Return (True, effective_price)
