## E2E Flow Verification

### Flow 1: Market Snapshot -> Strategy Evaluation -> Signal Emission

```
MarketSnapshot (Redis stream:market_snapshots)
  -> main.py deserialize_snapshot()
  -> FeatureStore.update(snapshot)
  -> classify_session(snapshot.timestamp)
  -> For each strategy in strategies_by_instrument:
     -> _apply_session_overrides(strategy, overrides)
     -> strategy.evaluate(snapshot, store)
     -> _restore_params(strategy, originals)
     -> For each signal:
        -> _apply_conviction_normalization(signal)
        -> publisher.publish(Channel.SIGNALS, signal_to_dict(signal))
```

**Status: COMPLETE** -- all steps are wired in main.py lines 351-417.

### Flow 2: Strategy Registration -> Matrix Filtering -> Instance Creation

```
STRATEGY_CLASSES dict (7 entries)
  -> build_strategies_for_instrument(instrument_id)
  -> load_strategy_matrix() filters by instrument
  -> validate_strategy_config() checks schema
  -> load_strategy_config_for_instrument() merges per-instrument overrides
  -> log_config_diff() logs parameter differences
  -> strategy_cls(config=config) instantiates
```

**Status: COMPLETE** -- wired in main.py lines 250-299.

### Flow 3: Funding Filter Integration

```
MarketSnapshot.funding_rate + hours_since_last_funding
  -> strategy.evaluate() calls compute_funding_boost()
  -> FundingBoostResult with z-score, boost, decay
  -> conviction += boost (for aligned directions)
```

**Status: COMPLETE** -- used by momentum (line 239), mean_reversion (line 285), correlation (line 180).

### Flow 4: Session-Aware Parameter Override

```
configs/sessions.yaml loaded at startup
  -> classify_session(timestamp) per snapshot
  -> get_session_overrides(config, strategy_name, session_type, instrument)
  -> _apply_session_overrides(strategy, overrides) mutates params
  -> strategy.evaluate() runs with overridden params
  -> _restore_params(strategy, originals) restores
```

**Status: COMPLETE** -- wired in main.py lines 371-396.

### Flow 5: Conviction Normalization + Portfolio A Routing

```
strategy.evaluate() -> signals with per-strategy conviction
  -> _apply_conviction_normalization(signal)
  -> normalize_conviction(conviction) -> band (low/medium/high)
  -> should_route_portfolio_a(conviction) -> true if >= 0.70
  -> signal updated with conviction_band metadata + PortfolioTarget.A
```

**Status: COMPLETE** -- wired in main.py lines 398-400.

### Flow 6: Adaptive Conviction Threshold Scaling

```
scipy.stats.percentileofscore (in adaptive_conviction.py)
  -> compute_adaptive_threshold(base, atr_array, current_atr)
  -> AdaptiveConvictionResult with scaled threshold
  -> Used by momentum, mean_reversion, regime_trend for vol-aware filtering
```

**Status: COMPLETE** -- 3 strategy consumers verified.

### Flow 7: Swing Point Detection

```
swing_points.py find_swing_low/find_swing_high
  -> Called by momentum.py for structure-aware stop placement
  -> Returns swing level or None (ATR fallback)
```

**Status: COMPLETE** -- momentum.py line 40 imports, uses in stop placement logic.
